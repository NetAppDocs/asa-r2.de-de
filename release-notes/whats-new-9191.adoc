[[san-configuration]]
== SAN-Konfiguration

[cols="2,4"]
|===
| Aktualisieren | Beschreibung 


| link:../learn-more/additional-supported-ontap-features.html#san-direct-attachment["Unterstützung für FC-Konfigurationen mit direkter Anbindung"] | Für die Verbindung von FC- und FC-NVMe-Hosts mit ONTAP Storage ist kein FC-Switch erforderlich. [[security]]
== Sicherheit

[cols="2,4"]
|===
| Aktualisieren | Beschreibung 


| link:../secure-data/enable-anti-ransomware-protection.html["Unterstützung für ARP/AI mit SnapMirror active sync-Beziehungen"] | Der autonome Ransomware-Schutz mit künstlicher Intelligenz (ARP/AI) unterstützt primäre Lese-/Schreibspeichereinheiten in SnapMirror Aktiv/Aktiv-Synchronisierungsbeziehungen auf ASA r2-Systemen. ARP/AI läuft auf den aktiven primären Speichereinheiten, erstellt ausschließlich primäre ARP-Snapshots und bietet auf maschinellem Lernen basierende Ransomware-Erkennung für diese Speichereinheiten. 


| link:../learn-more/additional-supported-ontap-features.html#tls-hardware-offload["Unterstützung für TLS-Hardware-Offload"] | Sie können die Transport Layer Security (TLS)-Verschlüsselung und -Entschlüsselung für Ihren gesamten Cluster auf unterstützte Ethernet-Karten auslagern. Dies reduziert die CPU-Auslastung und verbessert die Leistung von Systemen, die TLS verwenden. 
|===


[[storage-efficiency]]
== Storage-Effizienz

[cols="2,4"]
|===
| Aktualisieren | Beschreibung 


| link:../learn-more/additional-supported-ontap-features.html#global-deduplication["Unterstützung für globale Deduplizierung"] | Die globale Deduplizierung erweitert die bestehenden ONTAP Speicherplatzeinsparungen auf die gesamte Storage Availability Zone (SAZ). Durch das Entfernen doppelter Datenblöcke über alle Speichereinheiten in der SAZ hinweg kann die Speichereffizienz deutlich verbessert und die Gesamtbetriebskosten gesenkt werden. Die globale Deduplizierung ist standardmäßig aktiviert und arbeitet transparent, ohne dass eine administrative Konfiguration erforderlich ist. 
|===
